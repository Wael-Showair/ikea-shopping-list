Introduction

implementation file that provides all the needed functionalities for an animation controller to open or close a view as two sided door.

createSnapshots:afterScreenUpdates:

- (NSArray*)createSnapshots:(UIView*)view afterScreenUpdates:(BOOL) afterUpdates;
Parameters
view

the view from which snapshots are taken.

afterUpdates

A Boolean value that specifies whether the snapshot should be taken after recent changes have been incorporated.

Return Value

snapshots Array of two snapshots the left-hand side snapshot. of the view is at first index while the second index contains the right-hand side snapshot of the view.

Discussion

create a pair of snapshots from the given view.

transitionDuration:

-(NSTimeInterval)transitionDuration:(id<UIViewControllerContextTransitioning>)transitionContext;
Parameters
transitionContext

The context object containing information to use during the transition.

Return Value

The duration, in seconds, of the custom transition animation.

Discussion

UIKit asks the animator for the duration (in seconds) of the transition animation.

updateAnchorPoint:forView:

- (void)updateAnchorPoint:(CGPoint)anchorPoint forView:(UIView*)view;
Parameters
anchorPoint

location of the new anchor point.

view

the view object whose anchor point would be changed.

Discussion

Because the position in Core Animation is relative to the anchorPoint of the layer, changing the anchorPoint while maintaining the same position moves the layer. Compensation for such movement is dont with taking care of status/navigation bars as well.

Macro Definitions

FIRST_ANIMATION_START_TIME

Relative start time of the first animation which is rotation (in case of forward animation) or scale (in case of reserved animation).

ROTATION_ANIMATION_KEY_FRAME_DURATION

Total duration (in seconds) of the rotation animation.

SCALE_ANIMATION_KEY_FRAME_DURATION

Total duration (in seconds) of the scale up/down animation.

SECOND_ANIMATION_RELATIVE_START_TIME

Relative start time of the second animation. It ranges from 0 to 1.

The animation might be rotation (in case of forward animation) or scale (in case of reserved animation).

TRANSITION_DURATION

Total duration of the animation in seconds. It is summation of duration of scale animation keyframe and duration of rotation animation keyframe.
